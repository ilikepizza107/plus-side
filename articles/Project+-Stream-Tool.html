A Guide to the Newest Project+ Stream Tool Update

Written by ilikepizza107

Introduction

If you’ve tried to get into streaming Project+ in the last two years or so, you’ve probably seen or heard of my stream tool. While this old version of the tool was a great starting point for people, I had always wanted to go back and improve on it. Recently, this became a reality - and with this update, a whole slew of features were introduced. While I’d hope that the interface is clear enough for people to figure out what everything does, I understand that opening programs for the first time and seeing all the options can be overwhelming. That said, I wanted to take some time to go over the changes introduced in the newest version of the tool here.



Overview

This tool is made to be quick to set up and use. All it takes is dragging and dropping either the 16:9 or 4:3 .html files into OBS, launching the tool .exe, and you’re good to go.

The interface is designed to quickly change everything you’d need, including player names, pronouns, port colors, characters, scores, round, socials, and casters. I highly recommend checking out the linked video in the last sentence to get an idea of all the features the tool can do. You can add up to nine commentators (but for real, why are you going above five??), and the interface fully supports doubles!

The tool also has a VS Screen scene to be displayed when waiting for the next game, or to use as thumbnails for YouTube videos!

There’s also a Top 8 bracket scene to showcase your tournament’s top 8 positions. The scene has a transparent background, so you can use whatever background you feel fits the theme of your tournament.

Lastly, the stream tool has a Remote GUI feature, allowing for the tool to be accessed by any device within the local network, including mobile devices!

Every character and skin in Project+ is supported in the stream tool. However, if future Project+ updates add or remove skins, I will release an updated version of the tool with all of the current costumes!

Interface Shortcuts/hotkeys

Pressing Enter updates the scenes’ data.

Pressing either F1 or F2 to increase P1's or P2's score (or Team 1 or Team 2’s score, if in doubles mode).

Pressing ESC clears all player info.

For developing, there are some shortcuts to make things easier. Pressing F5 reloads the GUI, and pressing F12 opens the dev console. This will also unlock window resolution.

Feature Deep Dives

Tag (Sponsor) Logos

This update of the stream tool has a feature that allows you to quickly add logos or images to be placed on top of the overlay, depending on the tag a player is using. The images will be searched on the Resources/Logos/ folder. You can use the included example, typing +Side as a tag:

Since the sponsor possibilities could be infinite, there are no more images included as examples, so if you want to use this feature, you'll have to make your own images.

For 2v2, this feature will only work for team names, so in this case, if the team name is +Side, the logo will show up.

Player Presets

One of the bigger changes the tool has seen this update is the ability to add player and commentator presets. Simply enter in all the player information (including their character + skin, if making a player preset), then click the “Save preset and apply” button. The next time you go to enter that player’s name, you’ll be able to select that preset from the dropdown menu that appears, saving streamers precious time setting up for the next match! You’re also able to save presets with the same player name, allowing you to seamlessly and quickly switch between all of SipMastah’s four characters.

That being said, it's important to know where those are being stored so they can be shared or deleted, if desired. Player presets are stored in Resources/Texts/Player Info, and comms presets are stored in Resources/Texts/Caster Info.

Bracket Scene

In the bottom left-hand corner of the tool, there is a button to edit the top 8 bracket scene. Clicking on it will bring you to a different section of the GUI:

Clicking the top drop-down box will allow you to edit different parts of the bracket. While you could enter all the information manually, if you click the arrow button on the far right of a bracket section, it will import all player data that is already set up in the main part of the tool. Additionally, any player preset you’ve made will still work here - start typing their tag as usual and select the appropriate preset!

Remote GUI

As soon as you open the Stream Tool GUI, an HTTP server will be active, which means any device in your network that can open up a web browser, can access the Remote GUI. The Remote GUI functions exactly like it would on your computer, meaning all features of the tool will work no matter how you access it!

By default, you can access this remote GUI in the computer you're running the ST GUI with an url like 127.0.0.1:1111. First part of the url being the IP you want to access, and the second one being the port being used, separated by “:”. If you want to access the remote GUI from another device within the network, you will need to know the local IP of the computer that's running the ST GUI, then change 127.0.0.1 for that IP.

It's important to note that you are able to change the ports being used by the Stream Tool without having to rebuild the executable, something that could be useful to avoid colliding with other Stream Tools running in the same network. Over at Resources/Texts/GUI Settings.json, you will find the variables remoteUpdatePort and webSocketPort that can be changed to any value you need.

If you change webSocketPort however, this only changes the port that's used on the executable, so you will need to make sure to manually edit the port for the HTML files over at Resources/Scripts/Utils/Websocket.mjs for those to keep working (Ctrl+F 8080). This won't work for the Bracket however, but you can also Ctrl+F 8080 on Resources/Scripts/Bracket.js and change the number there.

OBS Transitions

2 basic transitions are included in the Resources/OBS Transitions folder, intended to be used to change to the game scene and to the vs screen, if you don't have a transition yourself of course. To use them on OBS:

  • Add a new stinger transition.
  • Set the video file to Game In.webm if creating the game scene transition, and Swoosh.webm if creating a vs screen transition.
  • Transition point -> 350 ms.
  • I recommend you to set the Audio Fade Style to crossfade, just in case.
  • On the scene's right click menu, set it to Transition Override to the transition you just created.

Additional Settings

In the bottom right-hand corner of the tool, there is a button to open the additional settings menu. Since some of these settings aren’t as straightforward, I’ll outline what they do here:

Allow Intro

Checking the “Allow Intro” button will make the scoreboard scenes use a special intro when transitioning to the scene.

Use Custom Round

This will allow you to input text instead of clicking an option from the round drop-down menu.

Force [W]/[L]

This allows you to set [W] and [L] for players in rounds. They will always appear when selecting Grand Finals/True Finals or typing “Grand” into the round box.

Update on score hotkey

Checking this will make scoreboard data instantly update when pressing a score hotkey (F1/F2).

Backwards score hotkeys

Score hotkeys (F1/F2) will remove score ticks instead of adding them.

Write simple texts

Saves plain text files at "Resources/Texts/Simple Texts/" to be used by other apps.

Match info to clipboard

Automatically copies match info to your clipboard when clicked, saving it as "Tournament Name - Round - Player1 (Character1) VS Player2 (Character2)".

Frequently Asked Questions

Q: How do I add my custom skins to this?

A: You’ll need the skin’s HD Character Select Portrait (CSP), and HD Stock Icon (STC). It is highly recommended that you do not use the SD versions of these textures. Next, go to Resources/Characters/(Character Name). In this example, I’ll be using Captain Falcon. The HD CSP will go into the “Skins” folder, and the HD STC will go into the “Icons” folder.

However, when you place the HD CSP in the Captain Falcon folder, you notice something…

That’s correct, some characters’ CSPs are flipped horizontally. To fix this, import the HD CSP into your favorite image editor (I use GIMP) and flip it. However, keep in mind that you will never have to flip STCs.

Next, rename the new CSP and STC to whatever you want, but keep in mind that they have to use the same name! I usually do “(Character Name) (Number)”. In this example, you can see I named this one “Captain Falcon (19)”. Remember this!

After you’ve renamed the CSP and STC, you’ll want to open the character’s “_Info.json” file. I use VSCode, but you can use Notepad or any text editor, really.

Finally, scroll down to the last entry of the “_Info.json” file, and add the appropriate lines, copying what has been done previously.

This is very important, but the last “}” in the skin list cannot have a comma after it. If it does, the .json file will break and it won’t work.

If everything has been done correctly, you should be able to see the added skin in the GUI!

Q: How do I add custom characters to this?

A: In Resources/Characters, create a new folder titled “(Character Name)”, for example “Ridley”. Then, you’ll need to add the following components: A “Icons” folder, a “Skins” folder, and an “_Info.json”. To make this as painless as possible, copy an existing character’s folder and delete all the CSPs and STCs from “Skins” and “Icons”. Then, follow the same procedure for adding skins as described above. You may have to edit some values in the bottom of the “_Info.json” to get the CSPs and STCs to display correctly in scenes, but with enough testing you should be able to get it looking how you want. Once you’re done, open the tool and your added character should be there.

Q: How do I customize the overlays or GUI?

A: This will require a decent amount of knowledge of how HTML, CSS, and JavaScript works. However, all CSS files are in the Resources/CSS folder, all JavaScript files are in the Resources/Scripts folder, and all .pngs that are used in the overlays are in the Resources/Overlay folder.
